--- layout: docs title: Links to the same place should be combined --- Because screen reader users often navigate web pages by jumping between links, it can be confusing if two links right next to each other, with different language, go to the same place. This kind of error can oftentimes be made when using WYSIWYG editors. For example:  

Please let us know if you need help!

<p>
  Please
  <a href="http://example.com/help">let us know</a>
  <a href="http://example.com/help">if you need help</a>!
</p>

Would read "let us know" and "if you need help" as two separate links, inferring different destinations, when they both go to the same place.

Guideline information:

This check applies to WCAG 2.0 Sections:
  • 2.4.4
  • 2.4.9
  • 4.1.2
and WCAG techniques:
  • H2
  • F89
